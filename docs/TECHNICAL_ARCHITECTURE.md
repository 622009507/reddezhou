## 系统概述

红德州是一个基于微服务架构的在线德州扑克游戏平台，采用前后端分离设计，支持高并发、高可用的游戏服务。

### 系统特点

- **微服务架构**：服务解耦，易于扩展和维护
- **前后端分离**：独立部署，灵活扩展
- **高性能**：基于Netty的异步非阻塞IO
- **高可用**：多副本部署，故障自动转移
- **可扩展**：水平扩展，支持百万级用户
- **安全性**：多层安全防护，保障数据安全

## 服务端架构

### 微服务架构

系统采用微服务架构，将不同的业务功能拆分为独立的服务：

#### 核心服务

1. **登录服务 (login)**
   - 端口：8500
   - 功能：用户认证、Token生成、权限验证
   - 技术：Netty + JWT
   - 数据库：MySQL

2. **房间服务 (room)**
   - 端口：8501
   - 功能：房间管理、玩家匹配、游戏逻辑
   - 技术：Netty + Redis
   - 数据库：MySQL + Redis

3. **锦标赛服务 (mtt)**
   - 端口：8502
   - 功能：多桌锦标赛、排名系统、重新买入
   - 技术：Netty + Redis
   - 数据库：MySQL + MongoDB

4. **奥马哈服务 (omaha)**
   - 端口：8503
   - 功能：奥马哈扑克、保险功能、Jackpot
   - 技术：Netty + Redis
   - 数据库：MySQL + MongoDB

5. **支付服务 (pay)**
   - 端口：8600
   - 功能：充值、提现、订单管理
   - 技术：Spring + MySQL
   - 数据库：MySQL

6. **资源服务 (res)**
   - 端口：8504
   - 功能：资源管理、版本更新、CDN
   - 技术：Spring + MongoDB
   - 数据库：MongoDB

7. **消息服务 (webmessage)**
   - 端口：8505
   - 功能：站内消息、推送通知
   - 技术：RabbitMQ + ActiveMQ
   - 数据库：MySQL

8. **运营后台 (yunying)**
   - 端口：8663
   - 功能：用户管理、数据统计、活动配置
   - 技术：Spring + Vue.js
   - 数据库：MySQL

9. **API服务 (webapi)**
   - 端口：8700
   - 功能：HTTP接口、数据查询
   - 技术：Spring + RESTful
   - 数据库：MySQL

10. **文件服务 (fileupload)**
    - 端口：8506
    - 功能：文件上传、图片处理
    - 技术：Spring + OSS
    - 数据库：MySQL

### 服务通信

#### 服务间通信方式

1. **RPC通信**
   - 使用Netty进行高性能RPC调用
   - 自定义协议，支持异步调用
   - 连接池管理，提高性能

2. **消息队列**
   - RabbitMQ：异步消息处理
   - ActiveMQ：系统消息通知
   - 解耦服务，提高系统可靠性

3. **HTTP接口**
   - RESTful API设计
   - 统一接口规范
   - 支持跨域访问

### 服务治理

#### 服务注册与发现

- **Zookeeper**：服务注册中心
- **服务注册**：服务启动时自动注册
- **服务发现**：动态获取服务列表
- **健康检查**：定期检查服务状态

#### 负载均衡

- **客户端负载均衡**：客户端选择服务实例

#### 熔断降级

- **熔断**：服务异常时自动熔断
- **降级**：提供降级方案
- **限流**：防止系统过载

#### 自定义协议

```
┌──────────┬──────────┬──────────┬──────────┬──────────┐
│  Header  │  Length  │  Opcode  │  Body    │  CRC     │
└──────────┴──────────┴──────────┴──────────┴──────────┘
    4字节      4字节      2字节      N字节      2字节
```

- **Header**：协议头，固定值"DZPK"
- **Length**：消息体长度
- **Opcode**：操作码，标识消息类型
- **Body**：消息体，使用Protobuf序列化
- **CRC**：校验码，保证数据完整性

## 总结

红德州采用现代化的微服务架构，具有以下特点：

1. **高可用**：多副本部署，故障自动转移
2. **高性能**：Netty异步IO，Redis缓存加速
3. **可扩展**：水平扩展，支持百万级用户
4. **安全性**：多层安全防护，保障数据安全
5. **易维护**：服务解耦，易于维护和扩展

该架构设计合理，技术选型先进，能够满足在线游戏平台的高并发、高可用需求。
